# /save — 落盘并准备开新对话

## 步骤

1. **识别 topic 文件**：回顾对话，找到涉及的 `memory/` topic 文件
2. **更新 topic 文件**（事实源）：有长期保留价值的信息才更新，先读全文整合，不追加堆砌
3. **写 daily memory**：追加到 `memory/_daily/YYYY-MM-DD.md`，精炼记录今天做了什么
4. **检查 MEMORY.md 指针**：仅新增 topic / 红线变更 / 路径变化时才更新，通常不动
5. **交接检查**：进行中但未写入 topic 的事 → 补写标注"待实施"或"待确认"
6. **文件大小检查**（`wc -c` 实际执行）：topic ≤ 3KB、MEMORY.md ≤ 2KB、daily ≤ 2KB，超限先精简
7. **回复确认**：
   > 已落盘：更新了 [文件列表]。
   > 变更摘要：[每个文件改了什么]
   > 下次开工提醒：[待续事项]
   > 可以 /new 了。

## 规则

- 无实质内容 → 「没什么需要落盘的，直接 /new 吧」
- MEMORY.md 绝不写具体数字，只放指针和状态词
- 防重复：5 分钟内已落盘且无新内容 → 「刚落过盘，没有新内容需要更新」
